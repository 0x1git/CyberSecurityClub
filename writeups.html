1. What is SQL Injection?

SQL Injection is a web security vulnerability that allows an attacker to interfere with the queries that an application makes to its database. It generally allows an attacker to view data that they are not normally able to retrieve.

SQL 

                SELECT * FROM users WHERE email = 'user@example.com';

Always use parameterized queries to prevent SQL injection attacks and validate all user inputs.

2. How to Secure APIs with JWT

JSON Web Tokens (JWTs) are a secure way to transmit information between parties as a JSON object. This information can be verified and trusted because it is digitally signed.

Node.js 

                const token = jwt.sign({ id: user._id }, process.env.JWT_SECRET, { expiresIn: "1h" });

Always store your secret keys safely and rotate JWTs periodically to enhance API security.

4. Basics of Hashing Passwords

Hashing passwords before storing them in a database is a fundamental security practice. Never store passwords in plain text.

Python 

                import bcrypt
                password = "mypassword"
                hashed = bcrypt.hashpw(password.encode('utf-8'), bcrypt.gensalt())

Always use strong hashing algorithms like bcrypt and include salt for additional security.

5. Difference Between var, let and const

JavaScript offers multiple ways to declare variables. Understanding the differences between var, let, and const is crucial for writing clean code.

JavaScript 

                var a = 10;    // Function scoped
                let b = 20;    // Block scoped
                const c = 30;  // Block scoped (constant)

Prefer using const for values that won't change, and let for variables that will be reassigned.

6. Cross-Site Scripting (XSS) Prevention

Cross-Site Scripting attacks occur when malicious scripts are injected into trusted websites. Learn how to prevent these attacks.

JavaScript 

                // Escape user input
                function escapeHtml(unsafe) {
                    return unsafe.replace(/[&<"']/g, function(m) {
                        return {'&':'&','<':'<','"':'"',"'":'''}[m];
                    });
                }

Always sanitize user inputs and use Content Security Policy (CSP) headers to prevent XSS attacks.

8. Cryptography Basics

Cryptography is the practice of securing information by transforming it into an unreadable format. Learn about encryption, decryption, and digital signatures.

Python 

                from cryptography.fernet import Fernet
                
                # Generate a key
                key = Fernet.generate_key()
                cipher_suite = Fernet(key)
                
                # Encrypt data
                encrypted_data = cipher_suite.encrypt(b"Hello World")

Use established cryptographic libraries and never implement your own encryption algorithms from scratch.
